## Project Overview

This is a __PROJECT_NAME__ API built with the [Gopernicus](https://gopernicus.com) framework.

## Architecture

Gopernicus organizes code into layers where alphabetical order defines the import rule:

| Layer          | Directory         | Can Import                           |
| -------------- | ----------------- | ------------------------------------ |
| App            | app/              | Everything (composition root)        |
| Bridge         | bridge/           | Core, Infrastructure, SDK            |
| Core           | core/             | Infrastructure, SDK                  |
| Infrastructure | infrastructure/   | SDK                                  |
| SDK            | sdk/              | Nothing from the framework           |
| Telemetry      | telemetry/        | SDK                                  |
| Workshop       | workshop/         | Not part of the deployed binary      |

If you see an import that violates this ordering, it is wrong.

## Code Generation

Generated files carry the header: // Code generated by gopernicus. DO NOT EDIT.
Never edit generated files directly. Instead:
- Data layer: modify annotations in queries.sql, then run gopernicus generate
- HTTP layer: modify bridge.yml, then run gopernicus generate
- Both: see workshop/documentation/gopernicus/ for full annotation and config reference

Bootstrap files (repository.go, bridge.go, routes.go, http.go, store.go) are created once and never overwritten. Add custom methods above the // gopernicus:start marker in Storer interfaces.

## Conventions

- Accept interfaces, return structs
- Variables and constants at the top of files, then interfaces, then structs
- Never abbreviate authentication as authn or authorization as authz
- Use the full words: authentication, authorization, authenticator, authorizer
- Package names: short, lowercase, no underscores
- Explicit dependency injection via constructors — no DI containers or service locators
- Functional options for optional configuration: Option func(*T)
- Error wrapping with fmt.Errorf and sentinel errors from sdk/errs

## Common Commands

```bash
gopernicus generate              # regenerate all domains
gopernicus generate <domain>     # regenerate a specific domain
gopernicus db create <name>      # create a new migration
gopernicus db migrate            # apply pending migrations
gopernicus db reflect            # reflect schema to JSON
gopernicus new repo <domain/entity>  # scaffold a new entity
gopernicus new case <name>       # scaffold a new use case
```

## Testing

```bash
# Unit tests
go test ./...

# Integration tests (requires running database)
go test -tags integration ./...
```
