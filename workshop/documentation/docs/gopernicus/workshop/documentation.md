# Workshop — Documentation

If you choose to document your application within the repository itself, `workshop/documentation/` is a good home for it — architecture notes, runbooks, onboarding guides, and the like. Keeping it close to the code means it tends to get updated alongside the changes that motivate it.

What you put in here, how you structure it, and whether you use it at all is entirely up to you.
