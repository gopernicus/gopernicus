# App Layer

The app layer is the composition root. It assembles infrastructure, core, and
bridge layers, wires all dependencies, and starts the server. This is the only
layer that imports from all other layers. It is generated by `gopernicus` and
lives in the user's project (not in the framework).

---

## Structure

A generated app has two key files:

| File | Purpose |
|------|---------|
| `main.go` | Entry point. Loads `.env`, configures logging, calls `run()`. |
| `app/server/config/server.go` | Composition root. Constructs all dependencies and returns a `Server`. |

---

## main.go -- Entry Point

The generated `main.go` follows a consistent pattern:

1. **Load environment** -- `environment.LoadEnv()` reads `.env` (missing file is not an error).
2. **Configure logger** -- `environment.ParseEnvTags(AppName, &logCfg)` parses
   `APPNAME_LOG_LEVEL`, `APPNAME_LOG_FORMAT`, etc. into `logger.Options`.
3. **Initialize infrastructure** -- each external dependency is configured from
   env vars and constructed:
   - **Database** -- `pgxdb.New(pgCfg)` creates a connection pool.
   - **Redis** (optional) -- `goredisdb.New(cfg)` for cache + event backends.
   - **Event bus** -- `memorybus.New(log)` or `goredisbus.New(rdb, log, cfg)`,
     optionally decorated with outbox for durable events.
   - **Cache** -- `memorycache.New()` or `rediscache.New(rdb)`, wrapped by `cache.New()`.
   - **Storage** (optional) -- `diskstorage.New()`, `gcs.New()`, or `s3.New()`,
     wrapped by `storage.New(log, client)`.
   - **Email** -- `stdoutemailer.New(log)` or `sendgridemailer.New(...)`,
     wrapped by `emailer.New(log, client, defaultFrom)`.
4. **Create server** -- passes all infrastructure to `server.New()`.
5. **Start and wait** -- runs the server in a goroutine, listens for SIGINT/SIGTERM,
   and calls `srv.Shutdown(ctx)` for graceful shutdown.

```go
srv, err := server.New(ctx, log,
    server.Config{AppName: AppName, Build: build},
    server.Infrastructure{
        Pool:     pool,
        EventBus: eventBus,
        Cache:    cacher,
        Emailer:  mailer,
    },
)
```

## server.go -- Composition Root

`server.New()` is where all dependency injection happens. The wiring follows a
strict top-down order:

### 1. Async Pool

```go
asyncPool := async.NewPool(
    async.WithMaxConcurrency(50),
    async.WithDropOnFull(true),
    async.WithLogger(log),
)
```

Shared bounded goroutine pool for fire-and-forget tasks. Drained during shutdown.

### 2. Rate Limiter

```go
rateLimiter := ratelimiter.New(memorylimiter.New(), ratelimiter.NewDefaultResolver(), log)
```

### 3. Repositories

Each domain's composite constructor creates all entity repositories with their
pgx stores, cache wrappers, and event bus:

```go
authRepos := auth.NewRepositories(log, infra.Pool, cacher, bus)
rebacRepos := rebac.NewRepositories(log, infra.Pool, cacher, bus)
```

### 4. Authorization

The authorization schema is composed from all domain schemas. The store chain
uses a satisfier adapter + cache decorator:

```go
authorizationSchema := authorization.NewSchema(authRepos.AuthSchema(), rebacRepos.AuthSchema())
authorizationStore := authorization.NewCacheStore(
    authorizationsatisfiers.NewAuthorizationStoreSatisfier(rebacRepos.RebacRelationship),
    cacher,
)
authorizer := authorization.NewAuthorizer(authorizationStore, authorizationSchema, cfg)
```

### 5. Authentication

Constructed with satisfier adapters that bridge generated repositories to the
authentication interfaces:

```go
authenticator := authentication.NewAuthenticator(
    authentication.NewRepositories(
        authenticationsatisfiers.NewUserSatisfier(authRepos.User),
        authenticationsatisfiers.NewPasswordSatisfier(authRepos.UserPassword),
        authenticationsatisfiers.NewSessionSatisfier(authRepos.Session),
        authenticationsatisfiers.NewVerificationTokenSatisfier(authRepos.VerificationToken),
        authenticationsatisfiers.NewVerificationCodeSatisfier(authRepos.VerificationCode),
    ),
    hasher, signer, bus, authCfg,
)
```

### 6. Web Handler and Middleware

```go
webHandler := web.NewWebHandler(web.WithLogging(log))
webHandler.Use(
    httpmid.TrustProxies(0),
    httpmid.Logger(log),
    httpmid.Panics(log),
)
```

Global middleware is applied to every request. Per-route middleware is added
by bridges during route registration.

### 7. Routes

Bridges are constructed with all their dependencies and register their routes
on a shared API group:

```go
api := webHandler.Group("/api/v1")
authBridges := authreposbridge.NewBridges(log, authRepos, rateLimiter, authenticator, authorizer)
authBridges.AddHttpRoutes(api)
```

### 8. OpenAPI

```go
webHandler.ServeOpenAPI("/openapi.json", web.OpenAPIInfo{
    Title:   "MyApp API",
    Version: "1.0.0",
}, authBridges.OpenAPISpec(), rebacBridges.OpenAPISpec())
```

### 9. HTTP Server

```go
httpServer := web.NewServer(serverCfg, web.WithHandler(webHandler))
```

## Server Lifecycle

**`Server.Start(ctx)`** -- starts listening. Blocking call.

**`Server.Shutdown(ctx)`** -- graceful shutdown in order:

1. Stop HTTP server (waits for in-flight requests)
2. Drain async pool (waits for background tasks)
3. Close event bus (waits for pending events)

---

## Async Patterns (Increasing Durability)

The generated app supports three levels of async processing:

| Pattern | Where | Durability | Use For |
|---------|-------|-----------|---------|
| `AsyncPool` | `server.AsyncPool` | None (fire-and-forget) | Cache invalidation, non-critical fanout |
| Event bus (inner) | `memorybus` or `goredisbus` | Lost on restart (memory) or at-least-once (Redis) | Domain events, email triggers |
| Outbox-decorated bus | `outbox.NewBus(inner, writer)` | Transactional (DB + bus) | Critical cross-service events |

## Configuration Pattern

All configuration is loaded from environment variables using the `AppName`
prefix and `environment.ParseEnvTags`:

```go
var cfg authentication.Config
environment.ParseEnvTags("MYAPP", &cfg)
// reads MYAPP_ACCESS_TOKEN_EXPIRY, MYAPP_REFRESH_TOKEN_EXPIRY, etc.
```

Struct tags define the env var suffix and default value:

```go
AccessTokenExpiry time.Duration `env:"ACCESS_TOKEN_EXPIRY" default:"30m"`
```

## Adding Custom Domains

To add a new domain to the composition root:

1. Wire repositories: `myRepos := mydomain.NewRepositories(log, pool, cacher, bus)`
2. Construct bridges: `myBridges := mydomainbridge.NewBridges(log, myRepos, rateLimiter, authenticator, authorizer)`
3. Register routes: `myBridges.AddHttpRoutes(api)`
4. Add OpenAPI spec: include `myBridges.OpenAPISpec()` in `ServeOpenAPI`

For custom use cases, wire cases between repositories and bridges:

```go
myCase := mycase.New(myRepos.Widget, authorizer, bus)
myBridge := mycasebridge.New(log, myCase, rateLimiter)
myBridge.AddHttpRoutes(api.Group("/cases"))
```
